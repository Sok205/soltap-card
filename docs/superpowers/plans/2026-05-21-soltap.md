# SolTap Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build SolTap — an open-source NFC-tappable on-chain business card for Solana — and demo it at Solana Accelerate (~2026-06-21).

**Architecture:** Monorepo with Next.js web app (Solana Pay tx-request + fee-sponsored Metaplex Core mint), Rust axum indexer (Helius LaserStream → SSE feed), and a reusable TS `card-sdk` package. Single `config.toml` drives both apps for fork-friendly self-hosting.

**Tech Stack:** TypeScript (Next.js 15 App Router, `@solana/kit`, `@metaplex-foundation/mpl-core`, vitest), Rust (axum, tokio, solana-client, yellowstone-grpc, sqlx+sqlite), pnpm workspaces, Vercel + Fly.io for deploy.

- [ ] **Step 4: Implement `src/mint.ts`**

```ts
import { TransactionBuilder, publicKey, signerIdentity, createNoopSigner, Umi } from "@metaplex-foundation/umi";
import { create } from "@metaplex-foundation/mpl-core";
import type { BuildMintArgs } from "./types";

export async function buildHandshakeMintIx(
  umi: Umi,
  args: BuildMintArgs
): Promise<TransactionBuilder> {
  const { ownerInfo, recipient, collection, asset, edition, event } = args;
  const name = `Handshake with ${ownerInfo.name} — #${edition}`;
  return create(umi, {
    asset: createNoopSigner(publicKey(asset)),
    collection: { publicKey: publicKey(collection) } as any,
    name,
    uri: args.metadataUri,
    owner: publicKey(recipient),
  });
}
```

> Note on `createNoopSigner`: the asset keypair will be added as a real signer by the caller (the web route serializes the partial-sign). Pure SDK doesn't hold keys.

- [ ] **Step 3: `scripts/init-collection.ts`**

```ts
import fs from "node:fs";
import path from "node:path";
import bs58 from "bs58";
import toml from "@iarna/toml";
import { createUmi } from "@metaplex-foundation/umi-bundle-defaults";
import { generateSigner, keypairIdentity, publicKey } from "@metaplex-foundation/umi";
import { createCollection, create } from "@metaplex-foundation/mpl-core";

async function main() {
  const cfgPath = path.resolve("config.toml");
  const cfg = toml.parse(fs.readFileSync(cfgPath, "utf8")) as any;
  if (cfg.collection?.collection_address) {
    console.log("Already initialized:", cfg.collection.collection_address);
    return;
  }

  const umi = createUmi(cfg.chain.rpc_url);
  const ownerSecret = bs58.decode(process.env.OWNER_UPDATE_AUTHORITY_KEYPAIR_B58!);
  const owner = umi.eddsa.createKeypairFromSecretKey(ownerSecret);
  umi.use(keypairIdentity(owner));

  const collection = generateSigner(umi);
  await createCollection(umi, {
    collection,
    name: `${cfg.owner.name} — Handshakes @ ${cfg.event.name}`,
    uri: cfg.collection?.collection_uri ?? "https://soltap.app/collection.json",
  }).sendAndConfirm(umi);

  const ownerCard = generateSigner(umi);
  await create(umi, {
    asset: ownerCard,
    name: `${cfg.owner.name} — Card`,
    uri: cfg.collection?.owner_card_uri ?? "https://soltap.app/owner-card.json",
    plugins: [
      {
        type: "Attributes",
        attributeList: [
          { key: "handshake_count", value: "0" },
          { key: "last_handshake_at", value: "0" },
        ],
      },
    ],
  }).sendAndConfirm(umi);

  cfg.collection = cfg.collection ?? {};
  cfg.collection.collection_address = collection.publicKey.toString();
  cfg.collection.owner_card_asset = ownerCard.publicKey.toString();
  fs.writeFileSync(cfgPath, toml.stringify(cfg));
  console.log("Initialized:");
  console.log("  collection:", cfg.collection.collection_address);
  console.log("  owner card:", cfg.collection.owner_card_asset);
}

main().catch((e) => { console.error(e); process.exit(1); });
```

import fs from "node:fs";
import path from "node:path";
import toml from "@iarna/toml";

export type Config = {
  owner: { name: string; role: string; bio?: string; x?: string; github?: string; email?: string; wallet: string };
  chain: { cluster: "devnet" | "mainnet-beta"; rpc_url: string; helius_laserstream_url?: string };
  collection: { collection_address: string; owner_card_asset: string };
  art: { template: string; storage: "arweave" | "data-uri" };
  event: { name: string };
};

let cached: Config | null = null;

impl Store {

        sqlx::migrate!("./migrations").run(&pool).await?;
        Ok(Self { pool })

    pub async fn insert(&self, e: &HandshakeEvent) -> Result<()> {
        sqlx::query(
            "INSERT OR IGNORE INTO handshakes(signature,slot,recipient,asset,edition,ts) VALUES (?,?,?,?,?,?)"

## Risks the executor should watch for

- `mpl-core` JS API shape may drift between versions; pin to a specific minor version after first green.
- Yellowstone proto types are verbose; conversion from `geyser::SubscribeUpdateTransaction` to a form decoder accepts is the trickiest part of Phase 3. Budget extra time.
- Phantom's iOS handling of `solana:` URIs from NFC has occasionally regressed; keep QR fallback on every printed card.
- Fee-payer wallet drain: monitor balance; set up a simple cron that alerts when balance < 0.3 SOL.
